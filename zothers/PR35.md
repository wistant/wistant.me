# Title: feat(core): Pure MDX Architectural Migration

### Narrative Synthesis

The overarching goal of this strategic refactoring was the complete eradication of the `content-collections` dependency loop. Relocating all Markdown-based articles (Blogs and Certifications) into a strictly native App Router paradigm ensures frictionless scalability, atomic route control, and profound performance boosts at the Vercel Edge. This shift inherently guarantees flawless SSG without the volatile runtime discrepancies caused by massive compiler libraries.

### Granular Modifications

- **[Dependency Layer]** : Dismantled `content-collections`, its configuration handlers, and Next.js plugins, streamlining the manifest securely.
- **[Blog Module]** : Restructured the legacy dual-language `.mdx` models into universally supported App Router `page.mdx` entries nested at `/blog/[slug]/page.mdx`.
- **[Certifications Module]** : Relocated all certification files to strictly obey the `/certifications/[slug]/page.mdx` design paradigm, discarding complex filtering.
- **[Layout Wrappers]** : Extracted previously tightly coupled designs into decoupled, asynchronous Layout Wrappers (`BlogPostLayout`, `CertLayout`) to reinstate UI aesthetics without `any` unsafe typescript casts.
- **[Registry Engine]** : Developed a micro fs-driven MDX metadata parser (`src/lib/mdx-registry.ts`) powered by **Zod schemas** for robust validation, replacing fragile regex with structured types.
- **[SEO & Sitemap]** : Overhauled `sitemap.ts` to natively iterate over filesystem slugs across all locales, ensuring 100% SEO coverage.
